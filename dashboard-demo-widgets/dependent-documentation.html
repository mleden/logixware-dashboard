Download and Install

For a no-fuss install follow these directions:
  1. Create libs directory in webroot of target server/domain (if it doesn't already exist).
  2. Copy latest release of logixware-dashboard from GitHub.
  3. Open archive (zip or tar.gz) and copy resulting logixware-dashboard directory to libs directory of target server/domain.
  4. Point your browser to yourdomain.com/libs/logixware-dashboard/dashboard-demo.html and verify that the "demo dashboard" renders without browser console errors.
  5. Copy libs/logixware-dashboard/dashboard-demo.html to libs/logixware-dashboard/my-dashboard.html.
  6. Start "building out your content" by modifying libs/logixware-dashboard/my-dashboard.html.

Logixware Dashboard Widgets and Features

Each widget of three architecture values:
  1. DEPENDENT
    2. The first is a widget that uses resources loaded by the enclosing page.
  2. INDEPENDENT
    3. The second is a widget that loads its own resources.
  3. EXTERNAL
    4. The third is a (pre-existing) site or page that, by definition, loads its own resources.

Because of the implications of loading resources, the second and third are rendered in an iframe. To convey this to the Logixware Dashboard Plugin, we employ a variation on the "standard" JSON format required by the plugin. In particular, JSON attributes that are "non-native to the plugin", must be placed in the JSON record's "metadata".

Add "custom widgets" to your own "widgets directory" (e.g., my-widgets), either by creating from scratch or by copying and modifying the demo widgets.
